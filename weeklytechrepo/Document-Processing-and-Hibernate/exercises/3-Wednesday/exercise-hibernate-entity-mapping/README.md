# Exercise: Hibernate Entity Mapping

## Overview

**Mode:** Implementation (Code Lab)
**Time Estimate:** 2 Hours

**Scenario:** You are building the data persistence layer for a new E-Commerce platform using Spring Boot and Hibernate.

## Your Goal

Create annotated Java entities (User, Address, Order) and verify that Hibernate's auto-DDL generates the correct relational database tables.

---

## Starter Code Setup

Navigate to the `/starter_code` directory. The `spring-boot-starter-data-jpa` and `h2` dependencies are configured. `application.properties` is set up to `create-drop` the schema and show the generated SQL.

---

## Core Tasks

### Task 1: The User Entity

Create a `User` class mapping to a `users` table:

- A Long `id` (Primary Key, Auto-generated Identity).
- A String `email` (Must be unique, cannot be null).
- A String `username` (Cannot be null).

### Task 2: The Address Entity

Create an `Address` class mapping to an `addresses` table:

- A Long `id`.
- A String `street`.
- A String `city`.
- A String `zipCode`.

### Task 3: The One-to-Many Relationship

We want a User to have many Addresses.

1. Add a `List<Address> addresses` field in the `User` class. Annotate it appropriately for a One-to-Many mapping. Use `mappedBy` so that Address owns the foreign key.
2. In the `Address` class, add a `User user` field. Annotate it as `@ManyToOne` with a `@JoinColumn(name = "user_id")`.

### Task 4: The Order Entity and Many-to-One

Create an `Order` class mapped to `orders`:

- A Long `id`.
- A Double `totalAmount`.
- A User `user` referenced via `@ManyToOne`. An order belongs to a user.

### Task 5: Testing the Mapping

1. Open the provided `DatabaseTestRunner.java` class (a `CommandLineRunner`).
2. Instantiate a `User`, two `Addresses`, and an `Order`. Link the objects together in Java.
3. Use the injected `UserRepository`, `AddressRepository`, and `OrderRepository` to save the entities.
4. Run the application!

---

## Definition of Done

- When you start the application, the console outputs `CREATE TABLE` statements for `users`, `addresses`, and `orders`.
- The `addresses` table has a `user_id` foreign key column.
- The `orders` table has a `user_id` foreign key column.
- The `email` column in the `users` table has a `UNIQUE` constraint generated by Hibernate.
- (Self-Check): Did you remember that `user` and `order` are reserved keywords in many SQL databases? Did you use the `@Table` annotation to name them something else like `app_users` and `app_orders`? (If not, fix the SQL syntax error!)
